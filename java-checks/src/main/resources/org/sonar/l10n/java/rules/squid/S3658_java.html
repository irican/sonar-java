当单元测试中的测试代码抛出异常时，测试本身就会失败。 因此，无需用 try - catch 结构来包围测试代码以检测故障。 相反，您只需将异常类型移动到方法签名即可。

当 catch 块中存在失败断言时，此规则会引发问题。

不符合规范的代码示例

@Test
public void testMethod() {
  try {
            //一些代码
  } catch (MyException e) {
    Assert.fail(e.getMessage());  //不符合规范的
  }
}

合规的解决方案

@Test
public void testMethod() throws MyException {
    //一些代码
}
