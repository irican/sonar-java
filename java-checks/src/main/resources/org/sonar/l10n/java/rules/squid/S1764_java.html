在二元运算符的两侧使用相同的值几乎总是错误的。在为逻辑运算符的情况下，它是复制/粘贴错误，因此是一个bug，或者它只是没用的代码，应该简化。 在为按位运算符和大多数二元数学运算符的情况下，在运算符的两侧具有相同的值会产生可预测的结果，并且应该简化。

不合规的代码示例

if ( a == a ) { // 总是 true
  doZ();
}
if ( a != a ) { // 总是 false
  doY();
}
if ( a == b && a == b ) { // 如果第一个是true，第二个也是
  doX();
}
if ( a == b || a == b ) { // 如果第一个是true，第二个也是
  doW();
}

int j = 5 / 5; //总是 1
int k = 5 - 5; //总是 0

c.equals(c); //总是 true

例外

  • 此规则忽略 *， +， 和 =。
  • 针对自身测试浮点值的特定情况对于 NaN 是有效测试，因此被忽略。
  • 类似地，左移1到1在位掩码的构造中是常见的，也被忽略。 
float f;
if(f != f) { //测试 NaN 值
  System.out.println("f is NaN");
}

int i = 1 << 1; // 合规
int j = a << a; // 不合规
