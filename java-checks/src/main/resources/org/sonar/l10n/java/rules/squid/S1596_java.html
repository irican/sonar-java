自从Java 5中引入泛型以来，建议使用泛型类型，例如 List<String> ，而不是使用原始类型， 例如 List。将原始类型分配给泛型类型是不安全的，并且会生成警告。Collections 类的旧 EMPTY_... 字段返回原始类型，而较新的 empty...() 方法返回通用类型。

不合规的代码示例

List<String> collection1 = Collections.EMPTY_LIST;  // 不合规
Map<String, String> collection2 = Collections.EMPTY_MAP;  // 不合规
Set<String> collection3 = Collections.EMPTY_SET;  // 不合规

合规的解决方案

List<String> collection1 = Collections.emptyList();
Map<String, String> collection2 = Collections.emptyMap();
Set<String> collection3 = Collections.emptySet();
