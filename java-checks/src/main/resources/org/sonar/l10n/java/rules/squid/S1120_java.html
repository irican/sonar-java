正确的缩进是提高代码可读性的简单而有效的方法。团队中开发人员之间的一致缩进也减少了对源控制系统的差异，使代码审查更容易。

当缩进与配置的值不匹配时，此规则会报告问题。仅报告不当缩进部分的第一行。

不合规的代码示例

缩进大小为2：

class Foo {
  public int a;
   public int b;   // 不合规，应该从第4列开始

...

  public void doSomething() {
    if(something) {
          doSomethingElse();  //不合规，应该从第6列开始
  }   // 不合规，应该从第4列开始
  }
}

合规的解决方案

class Foo {
  public int a;
  public int b;

...

  public void doSomething() {
    if(something) {
        doSomethingElse();
    }
  }
}
