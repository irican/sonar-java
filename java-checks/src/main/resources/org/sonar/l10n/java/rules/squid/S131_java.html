最后的 default 子句的目的是防御性编程。 该子句应采取适当行为，或包含适当的评论，说明为何不采取任何行为。

不合规的代码示例

switch (param) {  //缺少default子句
  case 0:
    doSomething();
    break;
  case 1:
    doSomethingElse();
    break;
}

switch (param) {
  default: // default子句应放在最后
    error();
    break;
  case 0:
    doSomething();
    break;
  case 1:
    doSomethingElse();
    break;
}

合规的解决方案

switch (param) {
  case 0:
    doSomething();
    break;
  case 1:
    doSomethingElse();
    break;
  default:
    error();
    break;
}

例外

如果 switch 参数是 Enum ， 并且如果在 case 语句中使用了此枚举（enum）的所有的常量， 则不需要 default 子句。

例如：

public enum Day {
    SUNDAY, MONDAY
}
...
switch(day) {
  case SUNDAY:
    doSomething();
    break;
  case MONDAY:
    doSomethingElse();
    break;
}
