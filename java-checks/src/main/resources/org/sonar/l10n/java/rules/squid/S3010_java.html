在构造函数中为 static 字段赋值可能在运行时导致不可靠的行为，因为它将更改该类的所有实例的值。

相反，删除字段的static修饰符，或者静态初始化它。

不合规的代码示例

public class Person {
  static Date dateOfBirth;
  static int expectedFingers;

  public Person(date birthday) {
    dateOfBirth = birthday;  // 不合规；现在每个人都有这个birthday
    expectedFingers = 10;  // 不合规
  }
}

合规的解决方案

public class Person {
  Date dateOfBirth;
  static int expectedFingers = 10;

  public Person(date birthday) {
    dateOfBirth = birthday;
  }
}
