在特定的指定路径上寻找

system 依赖项。 这大大降低了可移植性，因为如果部署工件 在没有像你的那样配置的环境中，你的代码将不起作用。

不符合规范的代码示例

<dependency>
  <groupId>javax.sql</groupId>
  <artifactId>jdbc-stdext</artifactId>
  <version>2.0</version>
  <scope>system</scope>  <!-- 不合规的 -->
  <systemPath>/usr/bin/lib/rt.jar</systemPath>  <!-- 搬除这个 -->
</dependency>
