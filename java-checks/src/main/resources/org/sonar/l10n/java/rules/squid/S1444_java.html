在没有声明字段为“final”的情况下声明它为“public”和“static”是不合理的。大多数情况下，这是在几个对象之间共享状态的组件（kludge）。 但是使用这种方法，任何对象都可以对共享状态执行任何操作，比如将它设置为 null。

不合规的代码示例

public class Greeter {
  public static Foo foo = new Foo();
  ...
}

合规的解决方案

public class Greeter {
  public static final Foo FOO = new Foo();
  ...
}

请参阅

  • MITRE, CWE-500 - Public Static Field Not Marked Final
  • CERT OBJ10-J. - Do not use public static nonfinal fields
