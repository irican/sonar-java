运算符优先级的规则很复杂。可能会导致错误。因此，在复杂的运算语句中应使用括号来表明意义。但是，这并不意味着应该在每次运算中平白地添加括号。

当根据下表组合使用 && and || 、在条件中一起使用赋值和相等（==）或关系运算符，以及其他运算符时，此规则会提出问题：

+, -, *, /, % <<, >>, >>> & ^ |
+, -, *, /, % x x x x
<<, >>, >>> x x x x
& x x x x
^ x x x x
| x x x x

不合规的代码示例

x = a + b - c;
x = a + 1 << b;  // 不合规

if ( a > b || c < d || a == d) {...}
if ( a > b && c < d || a == b) {...}  // 不合规
if (a = f(b,c) == 1) { ... } // 不合规；会先考虑==

合规的解决方案

x = a + b - c;
x = (a + 1) << b;

if ( a > b || c < d || a == d) {...}
if ( (a > b && c < d) || a == b) {...}
if ( (a = f(b,c)) == 1) { ... }
