将一个方法命名为 tostring， hashcode() 或 equal 是：

  • 排印形式中的一个bug。重写 toString， Object.hashCode() （注意camelCasing命名法）或 Object.equals （注意结尾的's'）意味着，并且应用程序不会按照预期行动。
  • 故意这样做的。但是这个名称会使其他开发人员感到困惑，他们可能没有注意到命名的差异，或者认为这是一个bug。

在这两种情况下，都应该重命名方法。

不合规的代码示例

public int hashcode() { /* ... */ }  // 不合规

public String tostring() { /* ... */ } // 不合规

public boolean equal(Object obj) { /* ... */ }  // 不合规

合规的解决方法

@Override
public int hashCode() { /* ... */ }

@Override
public String toString() { /* ... */ }

@Override
public boolean equals(Object obj) { /* ... */ }
