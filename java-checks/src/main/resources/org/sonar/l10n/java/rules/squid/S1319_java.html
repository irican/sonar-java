Java Collections API的目的是提供定义良好的接口层次结构，以便隐藏实现细节。

实现类必须用于实例化新集合，但理想情况下，实例化的结果应该存储在类型为Java集合接口（Java Collection interface）的变量中。

有一个实现类时，此规则会报告问题：

  • 从 public 方法返回。
  • 被接受为 public 方法的参数。
  • 公开为 public 成员。

不合规的代码示例

public class Employees {
  private HashSet<Employee> employees = new HashSet<Employee>();  // 不合规 - "employees" 应该有 "Set" 类型而不是 "HashSet"

  public HashSet<Employee> getEmployees() {                       // 不合规
    return employees;
  }
}

合规的解决方案

public class Employees {
  private Set<Employee> employees = new HashSet<Employee>();      // 合规

  public Set<Employee> getEmployees() {                           // 合规
    return employees;
  }
}
