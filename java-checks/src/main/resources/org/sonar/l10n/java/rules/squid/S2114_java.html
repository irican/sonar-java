将集合作为参数传递给集合自己的方法要么是一个错误(有意使用其他参数)，要么仅仅是无意义的代码。

此外，由于某些方法要求参数在执行期间保持不变，因此将集合传递给自身可能会导致未定义的行为。

不合规的代码示例

List <Object> objs = new ArrayList<Object>();
objs.add("Hello");

objs.add(objs); // 不合规； 如果 objs.hashCode() 调用，则StackOverflowException 
objs.addAll(objs); // 不合规； 未定义的行为
objs.containsAll(objs); // 不合规； 总是 true
objs.removeAll(objs); // 不合规； 令人困惑。 使用 clear() 代替
objs.retainAll(objs); // 不合规； NOOP
