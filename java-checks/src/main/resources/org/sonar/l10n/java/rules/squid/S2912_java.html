好的代码之所以好，是因为它清楚地向维护者传达了原始程序员的意图，正确地选择indexOf 方法可以帮助将代码从混乱变为清晰。

如果需要查看子字符串是否位于字符串中的某个特定点之外，可以测试 indexOf 子字符串与目标点之间的关系，或者可以使用 indexOf 的版本，该版本使用一个起始参数。 后者可以说更清楚，因为结果是用-1进行测试的，-1是一个很容易识别的“未找到”指标。

不合规的代码示例

String name = "ismael";

if (name.indexOf("ae") > 2) { // 不合规
  // ...
}

合规的解决方案

String name = "ismael";

if (name.indexOf("ae", 2) > -1) {
  // ...
}
