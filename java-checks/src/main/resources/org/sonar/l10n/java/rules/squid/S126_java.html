只要 if 语句后跟一个或多个 else if 语句，此规则就适用；最后的 else if 后面应该跟一个 else 语句。

最后的 else 语句的目的是防御性编程。

else语句应采取适当的操作或包含适当的注释，以确定不采取任何操作的原因。 这与在 switch 语句 中末尾的 default 子句的要求一致。

不合规的代码示例

if (x == 0) {
  doSomething();
} else if (x == 1) {
  doSomethingElse();
}

合规的解决方案

if (x == 0) {
  doSomething();
} else if (x == 1) {
  doSomethingElse();
} else {
  throw new IllegalStateException();
}

  • MISRA C:2012, 15.7 - All if...else if constructs shall be terminated with an else statement
  • CERT, MSC01-C. - Strive for logical completeness
  • CERT, MSC57-J. - Strive for logical completeness
