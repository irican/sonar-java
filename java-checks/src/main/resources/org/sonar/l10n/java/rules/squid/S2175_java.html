可以使用不正确类型的参数调用一对 Collection 方法，但这样做是没有意义的，可能是使用错误参数的结果。 当 List.contains 或 List.remove 的参数类型与用于列表声明的类型无关时，此规则将引发问题。

不合规的代码示例

List<String> list = new ArrayList<String>();
Integer integer = Integer.valueOf(1);

if (list.contains(integer)) {  // 不合规。总是 false.
  list.remove(integer); // 不合规。 list.add(integer) 不编译，所以这将始终返回false
}

请参阅

  • CERT, EXP04-J. - Do not pass arguments to certain Java Collections Framework methods that are a different type than the collection parameter type
