强烈建议 不要 将通配符类型用作返回类型。因为类型推断规则相当复杂，所以该API的用户不太可能知道如何正确地使用它。

让我们以返回 "List<? extends Animal>"的方法为例。有没有可能在这个清单上加上一只狗，一只猫，…… 我们根本就不知道。 编译器也不允许，这就是为什么它不允许这样的直接使用。通配符类型的使用应该仅限于方法参数。

当方法返回通配符类型时，此规则会报告问题。

不合规的代码示例

List<? extends Animal> getAnimals(){...}

合规的解决方案

List<Animal> getAnimals(){...}

或者

List<Dog> getAnimals(){...}
