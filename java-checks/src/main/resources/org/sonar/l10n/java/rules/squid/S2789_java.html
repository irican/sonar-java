Optional的概念是，当null可能导致错误时，将使用它。在某种程度上，它替换了null，当使用Optional 时，从调用中返回或接收null 应该没有问题。

不合规的代码示例

public void doSomething () {
  Optional<String> optional = getOptional();
  if (optional != null) {  // 不合规
    // 使用optional实现一些功能...
  }
}

@Nullable // 不合规
public Optional<String> getOptional() {
  // ...
  return null;  // 不合规
}

合规的解决方案

public void doSomething () {
  Optional<String> optional = getOptional();
  optional.ifPresent(
    // 使用optional实现一些功能...
  );
}

public Optional<String> getOptional() {
  // ...
  return Optional.empty();
}
