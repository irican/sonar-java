for 循环停止条件应该针对不变值测试循环计数器（即在每次循环迭代的开始和结束时都为真的值）。理想情况下， 这意味着在循环开始之前将停止条件设置为局部变量。

非恒定的停止条件效率稍低，难以理解和维护，并且可能导致将来引入错误。

此规则监控三种类型的非恒定停止条件：

  • 当循环计数器在 for 循环的主体中更新时
  • 停止条件取决于方法调用
  • 当停止条件取决于对象属性时，因为在执行循环期间这些属性可能会发生变化。

不合规的代码示例

for (int i = 0; i < 10; i++) {
  ...
  i = i - 1; // 不合规；计数器在循环体中更新
  ...
}

合规的解决方案

for (int i = 0; i < 10; i++) {...}

请参阅

  • MISRA C:2004, 13.6 - Numeric variables being used within a for loop for iteration counting shall not be modified in the body of the loop.
  • MISRA C++:2008, 6-5-3 - The loop-counter shall not be modified within condition or statement.
