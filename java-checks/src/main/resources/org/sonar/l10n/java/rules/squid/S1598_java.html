按照惯例，Java类的物理位置（源目录）及其逻辑表示（包）应保持同步。 因此，位于 "src/org/bar/Foo.java" 的Java文件应该包含 "org.bar"的包。

遗憾的是，Java编译器并未强制执行此约定，并且没有任何东西可以阻止开发人员将“Foo.java”类作为“com.apple”包的一部分，但这可能会降低类及其应用程序的可维护性。

类似地，将源代码放在名称中有圆点的文件夹中，而不是放在具有相同文件夹结构的文件夹中，会编译源代码，但在运行时会导致问题。 例如，带有 org.foo.bar 的包声明的代码放在 org/foo.bar 将被编译，但是类加载器将始终找到类将其放入基于包结构的文件夹， 因此希望源代码位于 org/foo/bar 文件夹中。 因此foo.bar 对源文件来说不是恰当的文件夹名。
