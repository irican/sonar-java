Getters和setter提供了一种通过提供 public 方法来强制封装的方法，这些方法提供对 private 字段的受控访问。但是在具有多个字段的类中，使用剪切和粘贴快速创建所需的getter和setter并不常见，这可能导致getter或setter访问错误的字段。

此规则在以下任何一种情况下都会出现问题：

  • setter不会使用相应的名称更新字段。
  • getter不会访问具有相应名称的字段。

不合规的代码示例

class A {
  private int x;
  private int y;

  public void setX(int val) { // 不合规的: 字段 'x' 没有更新
    this.y = val;
  }

  public int getY() { // 不合规的: 字段 'y' 未在返回值中使用
    return this.x;
  }
}

合规的解决方案

class A {
  private int x;
  private int y;

  public void setX(int val) {
    this.x = val;
  }

  public int getY() {
    return this.y;
  }
}
