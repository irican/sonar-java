如果未正确声明JUnit 3 TestCase 中的 suite 方法，那么程序不会使用它。 要想使用，则必须命名这样的方法 “suite”，没有参数，并且是 public static （公共静态的），而且必须返回 junit.framework.Test 或者 junit.framework.TestSuite

同样，如果没有正确大写，也会忽略未正确大写的 setUp 和 tearDown 方法。

不符合规范的代码示例

Noncompliant Code Example

Test suite() { ... }  // 不合规; 必须是公共静态的（public static）
public static boolean suite() { ... }  // 不合规; 错误的返回类型
public static Test suit() { ... }  // 不合规; 方法名称中的拼写错误
public static Test suite(int count) { ... } // 不合规; 必须是不带参数的(no-arg)

public void setup() { ... } // 不合规; 应该是 setUp
public void tearDwon() { ... }  // 不合规; 应该是 tearDown

合规解决方案

public static Test suite() { ... }
public void setUp() { ... }
public void tearDown() { ... }
