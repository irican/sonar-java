一个太长的参数列表意味着应该创建一个新的结构来包装这些大量的参数，或者表示该函数的功能太过复杂。

不合规的代码示例

使用最多4个参数：

public void doSomething(int param1, int param2, int param3, String param4, long param5) {
...
}

合规的解决方案

public void doSomething(int param1, int param2, int param3, String param4) {
...
}

例外

使用Spring的 @RequestMapping （以及相关的快捷方式注释，比如 @GetRequest）或者 @JsonCreator 注释的方法可能有许多参数，但可以进行封装。因此忽略了这些方法。
