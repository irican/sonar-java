当null测试中的相等运算符或其后的逻辑运算符被反转时，代码在解引用（dereference）对象之前会对其进行安全的null测试。 不幸的是，效果正好相反 - 对象经过空值测试，如果它为null，则会解引用（dereference） only ，从而导致一个保证的空指针解除引用。

不合规的代码示例

if (str == null && str.length() == 0) {
  System.out.println("String is empty");
}

if (str != null || str.length() > 0) {
  System.out.println("String is not empty");
}

合规的解决方案

if (str == null || str.length() == 0) {
  System.out.println("String is empty");
}

if (str != null && str.length() > 0) {
  System.out.println("String is not empty");
}
