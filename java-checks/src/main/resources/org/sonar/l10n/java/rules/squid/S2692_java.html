大多数对indexOf 值的检查都将其与-1进行比较，因为0是一个有效的索引。任何查找值 >0 的检查都会忽略第一个元素，这可能是一个bug。 如果只是为了检查String 或 List中是否包含值，那么可以考虑使用contains 方法。

当从String 或 List检索到的indexOf 值，根据>0对索引值进行测试时，该规则会引发一个问题。

不合规的代码示例

String color = "blue";
String name = "ishmael";

List<String> strings = new ArrayList<String> ();
strings.add(color);
strings.add(name);

if (strings.indexOf(color) > 0) {  // 不合规
  // ...
}
if (name.indexOf("ish") > 0) { // 不合规
  // ...
}
if (name.indexOf("ae") > 0) { // 不合规
  // ...
}

合规的解决方案

String color = "blue";
String name = "ishmael";

List<String> strings = new ArrayList<String> ();
strings.add(color);
strings.add(name);

if (strings.indexOf(color) > -1) {
  // ...
}
if (name.indexOf("ish") >= 0) {
  // ...
}
if (name.contains("ae") {
  // ...
}
