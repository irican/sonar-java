当对 Collection 处理的类型的超类型执行对 Collection 项的迭代时，此规则会引发问题。

不建议依赖 Object 或 Object 之间的任何类以及 Collection 处理的真实类。虽然它被语言所接受，但这种做法降低了代码的可读性，并强制降低 Collection 的项目以便能够在其上调用方法，同时在迭代中使用正确的类型使事情变得更加清晰和简单。

不符合规范的代码示例

public Collection<Person> getPersons() { ... }

for (Object item : getPersons()) { // 不符合规范
  Person person = (Person) item; //不符合规范; 需要向下转换为正确的类型才能使用“item”
  person.getAdress();
}

合规的解决方案

for (Person person : getPersons()) { // 符合规范
  person.getAddress() ;
}
