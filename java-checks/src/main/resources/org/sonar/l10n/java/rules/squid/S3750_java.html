Spring @Controller s， @Service s， @Repository 默认情况下有 singleton 范围， 意味着只有一个类的实例在应用程序中实例化。 为这些类类型中的一个定义任何其他范围将导致在创建和销毁新实例时不必要的流失。 在繁忙的Web应用程序中，这可能会导致服务器上出现大量不必要的额外负载。

当 @Scope 注释应用于 @Controller ， @Service 时，此规则会引发问题，或者 @Repository ，任何值，但“singleton”。 @Scope（“singleton”）是多余的，但是被忽略了。

不符合规范的代码示例

@Scope("prototype")  // 不合规的
@Controller
public class HelloWorld {

@Controller
public class HelloWorld {
