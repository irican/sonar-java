让一个具有相同名称的类成员（不区分大小写）作为其封闭类（enclosing class）是令人困惑的。当您考虑为类本身去命名类实例这样的常见做法时尤其如此。

最佳做法要求重命名与封闭类同名的任意字段或成员，以更好地描述它所代表或保留的类的特定方面。

不合规的代码示例

public class Foo {
  private String foo;

  public String getFoo() { }
}

Foo foo = new Foo();
foo.getFoo() // 这个返回什么?

合规的解决方案

public class Foo {
  private String name;

  public String getName() { }
}

//...

Foo foo = new Foo();
foo.getName()

例外

当字段的类型是包含类（containing class）并且该字段是静态时，允许单例像该类型一样命名，不报告问题

public class Foo {
  ...
  private static Foo foo;
  public Foo getInstance() {
    if(foo==null) {
      foo = new Foo();
    }
    return foo;
  }
  ...
}
