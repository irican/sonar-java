assert 不适合参数验证，因为可以在运行时在JVM中禁用断言，这意味着错误的操作设置将完全消除预期的检查。此外，asserts 会失败抛出 AssertionErrors，而不是抛出某种类型的 Exception. 抛出Errors 完全超出了预期的正常范围 正常程序中的catch/throw 行为。

当 public 方法使用 assert s中的一个或多个参数时，此规则会引发问题。

不合规的代码示例

 public void setPrice(int price) {
  assert price >= 0 && price <= MAX_PRICE;
  // 设定价格
 }

合规解决方案

 public void setPrice(int price) {
  if (price < 0 || price > MAX_PRICE) {
    throw new IllegalArgumentException("Invalid price: " + price);
  }
  // 设定价格
 }
