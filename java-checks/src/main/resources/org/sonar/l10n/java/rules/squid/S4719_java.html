JDK7引入了类 java.nio.charset.StandardCharsets 。它为所有可用于Java平台的每个实现都可用的字符集提供常量。

  • ISO_8859_1
  • US_ASCII
  • UTF_16
  • UTF_16BE
  • UTF_16LE
  • UTF_8

这些常数应优先于：

- 使用诸如“UTF-8”之类的字符串，其缺点是需要 catch / throw UnsupportedEncodingException ，实际上永远不会发生

- 使用Guava的 Charsets 类，自JDK7以来已经过时了

不符合规范的代码示例

try {
  byte[] bytes = string.getBytes("UTF-8"); // 不合规;使用String而不是StandardCharsets.UTF_8
} catch (UnsupportedEncodingException e) {
  throw new AssertionError(e);
}
// ...
byte[] bytes = string.getBytes(Charsets.UTF_8); //不合规;自JDK7以来Guava方式已经过时了

合规解决方案

byte[] bytes = string.getBytes(StandardCharsets.UTF_8)
