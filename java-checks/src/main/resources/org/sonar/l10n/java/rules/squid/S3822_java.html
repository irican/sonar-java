对 hibernate.hbm2ddl.auto 使用任何值但“validate”可能导致应用程序使用的数据库模式被更改，删除或清除所有数据。 简而言之，使用此属性是有风险的，并且只应在生产时使用“validate”选项，如果它被使用的话。

不符合规范的代码示例

<session-factory>
  <property name="hibernate.hbm2ddl.auto">update</property>  <!-- 不合规的 -->
</session-factory>

合规的解决方案

<session-factory>
  <property name="hibernate.hbm2ddl.auto">validate</property>  <!-- 合规的 -->
</session-factory>

或者

<session-factory>
  <!-- Property deleted -->
</session-factory>
