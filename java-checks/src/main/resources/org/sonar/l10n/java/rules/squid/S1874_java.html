一旦被弃用，就应该避免这些类、接口及其成员被使用、继承或扩展。弃用是一个警告，表明类或接口已经被取代，并且最终将被删除。 “弃用期”可以让你平稳地过渡到即将退休的老技术。

不合规的代码示例

/**
 * @deprecated  As of release 1.3, replaced by {@link #Fee}
 */
@Deprecated
public class Fum { ... }

public class Foo {
  /**
   * @deprecated  As of release 1.7, replaced by {@link #doTheThingBetter()}
   */
  @Deprecated
  public void doTheThing() { ... }

  public void doTheThingBetter() { ... }
}

public class Bar extends Foo {
  public void doTheThing() { ... } // 不合规； 不要重写已弃用的方法或明确将其标记为@Deprecated
}

public class Bar extends Fum {  // 不合规； Fum已被弃用

  public void myMethod() {
    Foo foo = new Foo();  // 好的; 该类未被弃用
    foo.doTheThing();  // 不合规； doTheThing 方法被弃用
  }
}

请参阅

  • MITRE, CWE-477 - Use of Obsolete Functions
  • CERT, MET02-J. - Do not use deprecated or obsolete classes or methods
