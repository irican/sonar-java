Throwable.printStackTrace(...) 将 Throwable 及其堆栈（stack）跟踪打印（print）到某个流。默认情况下， System.Err 流可能会无意中暴露敏感信息。

应使用记录器（Loggers）来打印 Throwable，因为它们有许多优点：

  • 用户可以轻松检索日志。
  • 日志消息的格式是统一的，允许用户轻松浏览日志。

当 printStackTrace 在没有参数的情况下使用，即当堆栈跟踪被打印到默认流时，该规则会报告一个问题。

不合规的代码示例

try {
  /* ... */
} catch(Exception e) {
  e.printStackTrace();        // 不合规
}

合规的解决方案

try {
  /* ... */
} catch(Exception e) {
  LOGGER.log("context", e);
}
