@ComponentScan 用于查找哪个Spring @Component beans (@Service 或 @Repository 或 Controller) 在类路径中可用，因此可以在应用程序上下文中使用它们。这是一个方便的功能，尤其是当您开始一个新项目时，它具有减慢应用程序启动时间的缺点，特别是当应用程序变大时（即：它引用一个大型JAR文件，或者它引用了大量的JAR文件或base-package引用了大量的.class文件。）

@ComponentScan 应该由 @Import加载的显式Spring bean列表替换.

此规则也考虑了接口@SpringBootApplication 因为它使用@ComponentScan进行了注释。

不符合规范的代码示例

@ComponentScan
public class MyApplication {
...
}

@SpringBootApplication
public class MyApplication {
...
}

合规解决方案

@Configuration
@Import({
        DispatcherServletAutoConfiguration.class,
        ErrorMvcAutoConfiguration.class,
        HttpEncodingAutoConfiguration.class,
        HttpMessageConvertersAutoConfiguration.class,
        MultipartAutoConfiguration.class,
        ServerPropertiesAutoConfiguration.class,
        PropertyPlaceholderAutoConfiguration.class,
        WebMvcAutoConfiguration.class
})
public class MyApplication {
...
}
