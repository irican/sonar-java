因为 Object 实现了 hashCode，所以任何Java类都可以放入哈希结构中。但是，定义 equals(Object) 而不是 hashCode() 的类不能真正散列，因为根据 equals 方法等效的实例可以返回不同的散列。

不合规的代码示例

public class Student {  // 不是 hashCode() 方法； 不是散列的
  // ...

  public boolean equals(Object o) {
    // ...
  }
}

public class School {
  private Map<Student, Integer> studentBody = // 到目前为止OK
          new HashTable<Student, Integer>(); // 不合规

  // ...

合规的解决方案

public class Student {  // 有 hashCode() 方法； 可散列的
  // ...

  public boolean equals(Object o) {
    // ...
  }
  public int hashCode() {
    // ...
  }
}

public class School {
  private Map<Student, Integer> studentBody = new HashTable<Student, Integer>();

  // ...
