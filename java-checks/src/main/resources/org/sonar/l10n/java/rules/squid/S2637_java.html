字段、参数和返回值被标记为 @NotNull，@NonNull，或@Nonnull， 这些字段、参数和返回值被认为是非空值，通常在使用前不进行空检查。 因此，将这些值中的一个设置为null，或者未能在构造函数中设置此类类字段，可能会在运行时导致NullPointerException。

不合规的代码示例

public class MainClass {

  @Nonnull
  private String primary;
  private String secondary;

  public MainClass(String color) {
    if (color != null) {
      secondary = null;
    }
    primary = color;  // 不合规; "primary"是非空的，但可以在这里设置为空
  }

  public MainClass() { // 不合规; "primary" 非空，但未初始化
  }

  @Nonnull
  public String indirectMix() {
    String mix = null;
    return mix;  // 不合规; 返回值为非null，但返回null。}}
  }
