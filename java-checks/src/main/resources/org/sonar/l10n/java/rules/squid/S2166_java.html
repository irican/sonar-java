清晰，沟通性的命名在代码中很重要。它可以帮助维护者和API用户理解代码单元的意图和使用。在不扩展 Exception 或其子类之一的类的名称中使用“exception”明显违反了类的名称将指示它是什么和/或做什么的期望。

不合规的代码示例

public class FruitException {  // 不合规； 这与 Exception 无关
  private Fruit expected;
  private String unusualCharacteristics;
  private boolean appropriateForCommercialExploitation;
  // ...
}

public class CarException {  // 不合规； extends子句被遗忘了吗？
  public CarException(String message, Throwable cause) {
  // ...
