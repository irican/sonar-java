如果布尔表达式不改变条件的计算值，那么它完全没有必要，可以删除。如果这是无意义的，且因为它不符合程序员的意图，那么它是一个错误，应该修复表达式。

不合规的代码示例

a = true;
if (a) { // 不合规
  doSomething();
}

if (b && a) { // 不合规; "a" 总是为"true"
  doSomething();
}

if (c || !a) { // 不合规; "!a" 总是 "false"
  doSomething();
}

合规的解决方案

a = true;
if (foo(a)) {
  doSomething();
}

if (b) {
  doSomething();
}

if (c) {
  doSomething();
}

  • MITRE, CWE-570 - Expression is Always False
  • MITRE, CWE-489 - Leftover Debug Code
  • CERT, MSC12-C. - Detect and remove code that has no effect or is never executed
