在 String 对象上调用方法 toLowerCase()， toUpperCase() 或 format() 时未能指定区域设置（locale）， 这意味着将使用系统默认编码，可能会产生国际字符或数字表示的问题。例如，使用土耳其语，当将小写字母“i”转换为大写字母时，结果是大写字母“I”，上面有一个点。

没有区域设置（locale）的case转换在其“home”环境中可能工作得很好，但是对于使用不同编码的客户来说，这种转换的方式非常难以诊断。 当修复这些bug的时候，即使不能完全修复，也几乎不可能重现这些bug。对于对语言环境敏感的字符串，应该始终使用正确的语言环境， 但是 Locale.ENGLISH 可用于不区分大小写的语言环境。

不合规的代码示例

myString.toLowerCase()

合规的解决方案

myString.toLowerCase(Locale.TR)
