抽象类的目的是提供一些可继承的行为，同时定义必须由子类实现的方法。

没有抽象方法的类纯粹是为了防止实例化而被抽象化，应该用私有构造函数转换为具体类（即删除 abstract 关键字）。

应将只具有抽象方法但无可继承行为的类转换为接口。

不合规的代码示例

public abstract class Animal {  // 不合规； 应该是一个接口
  abstract void move();
  abstract void feed();
}

public abstract class Color {  // 不合规； 应该是具体的有一个私有构造函数
  private int red = 0;
  private int green = 0;
  private int blue = 0;

  public int getRed() {
    return red;
  }
}

合规的解决方案

public interface Animal {
  void move();
  void feed();
}

public class Color {
  private int red = 0;
  private int green = 0;
  private int blue = 0;

  private Color () {}

  public int getRed() {
    return red;
  }
}

public abstract class Lamp {

  private boolean switchLamp=false;

  public abstract void glow();

  public void flipSwitch() {
    switchLamp = !switchLamp;
    if (switchLamp) {
      glow();
    }
  }
}
