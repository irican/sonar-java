使用等于运算符（== 和 !=）检验 for 循环结束是危险的，因为它会产生一个无限循环。 应使用更广泛的关系运算符，从而不易写出无限循环（但并非不可能）。

不合规的代码示例

for (int i = 1; i != 10; i += 2)  // 不合规。无穷的；i从9直接到11
{
  //...
}

合规的解决方案

for (int i = 1; i <= 10; i += 2)  // 合规
{
  //...
}

例外

如果循环计数器未在循环体内修改，则忽略等于（==或！=）运算符，并且：

  • 开始值小于结束值，并在每次迭代时递增1。
  • 开始值大于结束值，并在每次迭代时减1。

当使用 null 检验循环终止时，也会忽略等于（==或！=）运算符。

for (int i = 0; arr[i] != null; i++) {
  // ...
}

for (int i = 0; (item = arr[i]) != null; i++) {
  // ...
}
