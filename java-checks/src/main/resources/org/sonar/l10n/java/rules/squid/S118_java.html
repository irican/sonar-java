共享一些命名约定是使团队有效协作的关键点。此规则允许检查所有 abstract 类名是否与提供的正则表达式匹配。 如果 非抽象类（non-abstract class）与正则表达式匹配，此规则也会提出问题，建议将其设为抽象或重命名。

不合规的代码示例

使用默认的正则表达式：^Abstract[A-Z][a-zA-Z0-9]*$：

abstract class MyClass { // 不合规
}

class AbstractLikeClass { // 不合规
}

合规的解决方案

abstract class MyAbstractClass {
}

class LikeClass {
}
