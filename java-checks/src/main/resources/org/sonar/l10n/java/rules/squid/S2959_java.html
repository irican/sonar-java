在“代码越干净越好”的理论下，应该省略try-with-resources构造的末尾的分号，因为它可以省略。

不合规的代码示例

try (ByteArrayInputStream b = new ByteArrayInputStream(new byte[10]);  // 忽略;这是必需的
      Reader r = new InputStreamReader(b);)   // 不合规
{
   //实现点功能
}

合规的解决方案

try (ByteArrayInputStream b = new ByteArrayInputStream(new byte[10]);
      Reader r = new InputStreamReader(b))
{
   //实现点功能
}
