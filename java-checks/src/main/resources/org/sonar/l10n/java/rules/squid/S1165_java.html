异常用于表示应用程序在发生错误时的状态。

这能确保在一个 Exception 类中所有的 final 字段都有以下优点:

  • 将在实例化 Exception 时完全定义。
  • 不会被可疑的错误处理程序更新或损坏。

这将使开发人员能够快速了解出现了什么问题。

不合规的代码示例

public class MyException extends Exception {

  private int status;                               // 不合规
  public MyException(String message) {
    super(message);
  }

  public int getStatus() {
    return status;
  }

  public void setStatus(int status) {
    this.status = status;
  }

}

合规的解决方案

public class MyException extends Exception {

  private final int status;

  public MyException(String message, int status) {
    super(message);
    this.status = status;
  }

  public int getStatus() {
    return status;
  }

}
