文件的导入部分应由集成开发环境（IDE）处理，而不是由开发人员手动处理。

如果是这种情况，则不应出现未使用和无用的导入。

将它们留在代码中会降低其可读性，因为它们的存在可能引起混淆。

不合规的代码示例

package my.company;

import java.lang.String;        // 不合规； 始终隐式导入java.lang类
import my.company.SomeClass;    // 不合规； 始终隐式导入同包文件
import java.io.File;            // 不合规； 文件未使用

import my.company2.SomeType;
import my.company2.SomeType;    // 不合规； 'SomeType' 已经导入

class ExampleClass {

  public String someString;
  public SomeType something;

}

例外

注释中提到的类型的导入（例如Javadocs）将被忽略。
