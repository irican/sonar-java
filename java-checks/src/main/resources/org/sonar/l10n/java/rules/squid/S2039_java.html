如果没有显式地声明成员变量的可见性，可能会导致它具有您意料之外的可见性，并且可能会让其他类对其进行意外的修改。

不合规的代码示例

class Ball {
    String color="red";  // 不合规
}
enum A {
  B;
  int a;
}

合规的解决方案

class Ball {
    private String color="red";  // 合规
}
enum A {
  B;
  private int a;
}

例外

使用 Guava 的 @VisibleForTesting 注解（annotation）来注解的成员将被忽略，因为它表明已经有意放宽了可见性以使代码可测试。

class Cone {
  @VisibleForTesting
  Logger logger; // 合规
}
