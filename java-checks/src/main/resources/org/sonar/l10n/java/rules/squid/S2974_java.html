只有private 构造函数的类应该标记为final ，以防止任何错误的扩展尝试。

不合规的代码示例

public class PrivateConstructorClass {  // 不合规
  private PrivateConstructorClass() {
    // ...
  }

  public static int magic(){
    return 42;
  }
}

合规的解决方案

public final class PrivateConstructorClass {  // 合规的
  private PrivateConstructorClass() {
    // ...
  }

  public static int magic(){
    return 42;
  }
}
