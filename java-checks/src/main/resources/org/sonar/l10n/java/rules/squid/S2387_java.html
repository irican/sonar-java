在两个不相关的类中使用相同名称的变量是可以的，但是在类层次结构中做同样的事情，你最多会感到困惑，最糟糕的是混乱。

不合规的代码示例

public class Fruit {
  protected Season ripe;
  protected Color flesh;

  // ...
}

public class Raspberry extends Fruit {
  private boolean ripe;  // 不合规的
  private static Color FLESH; // 不合规的
}

合规的解决方案

public class Fruit {
  protected Season ripe;
  protected Color flesh;

  // ...
}

public class Raspberry extends Fruit {
  private boolean ripened;
  private static Color FLESH_COLOR;

}

例外

此规则忽略父类和子类中 static 的同名字段。 此规则忽略 private 父类字段，但在所有其他此类情况下，应重命名子类字段。

public class Fruit {
  private Season ripe;
  // ...
}

public class Raspberry extends Fruit {
  private Season ripe;  // 无论如何，从Raspberry中看不到父字段'成熟'的合规性
  // ...
}
