if 语句的条件始终为false，则会使代码块失去功能。 if语句的条件始终为true，则完全是冗余的，这会降低代码的可读性。

存在此类代码有三种可能的原因：

  • 调试期间更改了if语句，并且已提交调试代码。
  • 有些值没有设置。
  • 有些逻辑并没有做程序员认为它做的事情。

在任何这些情况下，应删除条件不恰当的 if 语句。

不合规的代码示例

if (true) {
  doSomething();
}
...
if (false) {
  doSomethingElse();
}

if (2 < 3 ) { ... }  // 不合规；总是 false

int i = 0;
int j = 0;
// ...
j = foo();

if (j > 0 && i > 0) { ... }  // 不合规；总是 false - 在初始化后i永远不会被处理

boolean b = true;
//...
if (b || !b) { ... }  // 不合规

合规的解决方案

doSomething();
...

请参阅

  • MITRE, CWE-570 - Expression is Always False
  • MITRE, CWE-571 - Expression is Always True
  • MISRA C:2004, 13.7 - Boolean operations whose results are invariant shall not be permitted.
  • MISRA C:2012, 14.3 - Controlling expressions shall not be invariant
