仅仅因为你可以做某事，并不意味着你应该这样做，嵌套三元操作就是这种情况。嵌套三元运算符会产生一种代码，这些代码在您编写代码时可能看起来很清楚，但六个月之后会让维护者（或者更糟 - 让未来的您）挠头和难受。

相反，在清晰度方面犯了错误，并使用另一行将嵌套操作表示为单独的语句。

不合规的代码示例

public String getTitle(Person p) {
  return p.gender == Person.MALE ? "Mr. " : p.isMarried() ? "Mrs. " : "Miss ";  // 不合规的
}

合规解决方案

public String getTitle(Person p) {
  if (p.gender == Person.MALE) {
    return "Mr. ";
  }
  return p.isMarried() ? "Mrs. " : "Miss ";
}
