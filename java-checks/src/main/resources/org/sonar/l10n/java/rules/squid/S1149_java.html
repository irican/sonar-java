Java API的早期类，比如 Vector， Hashtable 和 StringBuffer，已经过同步以使它们具有线程安全性（thread-safe）。 但不幸的是，即使从单个线程使用这些集合，同步（synchronization）也会对性能产生很大的负面影响。

最好使用新的非同步替换（unsynchronized replacements）：

  • 使用 ArrayList 或 LinkedList 而不是 Vector
  • 使用Deque 而不是 Stack
  • 使用HashMap 而不是 Hashtable
  • 使用StringBuilder 而不是 StringBuffer

不合规的代码示例

Vector cats = new Vector();

合规的解决方案

ArrayList cats = new ArrayList();

例外

在重写方法（overriding method）的签名中忽略这些同步类的使用。

@Override
public Vector getCats() {...}
