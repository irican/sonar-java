操作符对的使用( =+，=- 或 =! ) 顺序被反转，则表示单个操作符(+=，-= 或 !=) 将编译并运行，但不会产生预期的结果。

当=+， =-， or =!在两个操作符之间没有任何间距的情况下使用，且后面至少有一个空格字符时，此规则会引发一个问题。

不合规的代码示例

int target = -5;
int num = 3;

target =- num;  // 不合规； target = -3。这是什么意思。
target =+ num; // 不合规； target = 3

合规的解决方案

int target = -5;
int num = 3;

target = -num;  // 合理的;分配num逆值的意图很明显
target += num;
