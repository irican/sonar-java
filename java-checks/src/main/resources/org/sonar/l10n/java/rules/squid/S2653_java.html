在web应用程序中没有理由使用main 方法。它可能对应用程序开发期间的调试很有用，但是这种方法不应该投入生产。 在web应用程序中使用main 方法打开了一扇通往应用程序逻辑的大门，攻击者可能永远无法触及该逻辑(但是要注意如果攻击者触及了该逻辑)，但是这是一种草率的做法，并表明可能存在其他问题。

当在servlet或EJB中发现main方法时，该规则会引发一个问题。

不合规的代码示例

public class MyServlet extends HttpServlet {
  public void doGet(HttpServletRequest req, HttpServletResponse res) throws ServletException, IOException {
    if (userIsAuthorized(req)) {
      updatePrices(req);
    }
  }

  public static void main(String[] args) { // 不合规
    updatePrices(req);
  }
}

请参阅

  • MITRE, CWE-489 - Leftover Debug Code
  • OWASP Top 10 2017 Category A3 - Sensitive Data Exposure
  • CERT, ENV06-J. - Production code must not contain debugging entry points
