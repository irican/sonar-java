Java语言的作者非常坦率地说 Optional 仅用作返回类型，即作为一个可能返回值或可能不返回值的传达方法。

为此，它很有价值，但在输入端使用 Optional 会增加您在方法中必须完成的工作，而不会真正增加值。使用 Optional 参数，您可以从两个可能的输入：null和not-null，到三个：null，非null-without-value和非null-with-value。除此之外，长期可以使用重载来传达某些参数是可选的，并且没有理由拥有 Optional 参数。

不符合规范的代码示例

public String sayHello(Optional<String> name) {  // 不合规的
  if (name == null || !name.isPresent()) {
    return "Hello World";
  } else {
    return "Hello " + name;
  }
}

合规解决方案

public String sayHello(String name) {
  if (name == null) {
    return "Hello World";
  } else {
    return "Hello " + name;
  }
}
