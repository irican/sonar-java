NullPointerException 应该表示意外遇到了一个 null 值。良好的编程实践要求代码结构化来避免NPE。

显式抛出 NullPointerException 迫使方法的调用者显式捕获它，而不是编码来避免它。此外， 很难区分意外遇到的 null 值和导致方法故意抛出NPE的条件。

如果抛出NPE以指示方法的参数不应为null，请使用 @NotNull 注解（annotation）

不合规的代码示例

public void doSomething (String aString) throws NullPointerException {
     throw new NullPointerException();
}

合规的解决方案

public void doSomething (@NotNull String aString) {
}
