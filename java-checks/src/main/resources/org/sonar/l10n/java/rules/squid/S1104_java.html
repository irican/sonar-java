公共类变量字段不遵循封装原则，并且有三个主要缺点：

  • 无法添加其他行为，例如验证（validation）。
  • 内部表示（internal representation）是暴露的，以后不能更改。
  • 成员值可能会在代码中的任何位置发生变化，并且可能程序员所想的不一样。

通过使用私有属性和访问器（accessor）方法（set和get），可以防止未经授权的修改。

不合规的代码示例

public class MyClass {

  public static final int SOME_CONSTANT = 0;     // 合规 - 不检查常量

  public String firstName;                       // 不合规

}

合规的解决方案

public class MyClass {

  public static final int SOME_CONSTANT = 0;     // 合规 - 不检查常量

  private String firstName;                      // 合规

  public String getFirstName() {
    return firstName;
  }

  public void setFirstName(String firstName) {
    this.firstName = firstName;
  }

}

例外

因为它们不可修改，所以此规则忽略 public final 字段。
