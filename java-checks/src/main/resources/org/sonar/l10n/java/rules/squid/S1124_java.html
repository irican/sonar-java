Java语言规范建议按以下顺序列出修饰符：

1. Annotations

2. public

3. protected

4. private

5. abstract

6. static

7. final

8. transient

9. volatile

10. synchronized

11. native

12. strictfp

不遵循此约定没有技术影响，但会降低代码的可读性，因为大多数开发人员习惯于标准顺序。

不合规的代码示例

static public void main(String[] args) {   // 不合规
}

合规的解决方案

public static void main(String[] args) {   // 合规的
}
