使用已检查异常（checked exception）来强制方法调用程序通过传播错误或处理错误去解决错误。抛出异常使它们完全成为方法API的一部分。

但是为了使调用程序保持合理的复杂性，方法不应该抛出超过一种已检查异常。

不合规的代码示例

public void delete() throws IOException, SQLException {      // 不合规
  /* ... */
}

合规的解决方案

public void delete() throws SomeApplicationLevelException {
  /* ... */
}

例外

此规则不检查重写方法，允许抛出几个已检查的异常。
