基本身份验证惟一的混淆方法是Base64编码。由于Base64编码很容易识别和逆转，所以它只向用户提供了最薄弱的保护，不应该使用它。

不合规的代码示例

// 从Apache HttpClient使用 HttpPost
String encoding = Base64Encoder.encode ("login:passwd");
org.apache.http.client.methods.HttpPost httppost = new HttpPost(url);
httppost.setHeader("Authorization", "Basic " + encoding);  // 不合规

or

// 使用 HttpURLConnection
String encoding = Base64.getEncoder().encodeToString(("login:passwd").getBytes(‌"UTF‌​-8"​));
HttpURLConnection conn = (HttpURLConnection) url.openConnection();
conn.setRequestMethod("POST");
conn.setDoOutput(true);
conn.setRequestProperty("Authorization", "Basic " + encoding); // 不合规
