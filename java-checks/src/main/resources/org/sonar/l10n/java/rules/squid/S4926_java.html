Serializable 强烈建议在 Serializable 类上提供 serialVersionUID 字段 文档，但盲目地遵循该建议可能是有害的。

serialVersionUID 值与序列化数据一起存储，并且在反序列化数据时验证该字段，以确保读取数据的代码与序列化数据兼容。如果失败，这意味着序列化数据和代码不同步，这很好，因为你知道什么是错的。

当 serialVersionUID 由IDE生成或盲目硬编码时，很有可能在时忘记更新 serialVersionUID 值可序列化的类稍后会增加其他字段。因此，旧的序列化数据将被错误地认为与较新版本的代码兼容，从而导致难以调试的情况。

因此，应谨慎定义 serialVersionUID 。此规则在每个 serialVersionUID 字段上引发问题，在实现 Serializable 的类上声明，以确保团队对 serialVersionUID 字段的存在和值进行质询和验证。

不符合规范的代码示例

public class Foo implements Serializable {
  private static final long serialVersionUID = 1;
}

public class BarException extends RuntimeException {
  private static final long serialVersionUID = 8582433437601788991L;
}
