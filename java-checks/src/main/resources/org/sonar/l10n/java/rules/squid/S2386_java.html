没有充分理由让一个可变对象成为public （默认情况下），static （静态接口）的成员interface. 应将这些变量移入类中，降低其可见性。

同样，应该尽可能地保护直接访问而不是通过getter和setter访问的类和枚举的可变 static（静态） 成员。 这可以通过降低可见性或在适当的情况下写字段 final 来完成。

请注意，创建一个可变字段，例如数组（Final）， final 将不会重新分配变量，但这样做不会影响数组内部状态的可变性(也就是说，它无法实现目标)。

此规则为public static （公共静态数组）, Collection（集合）, Date(日期),和awt.Point 成员带来问题。

不合规的代码示例

public interface MyInterface {
  public static String [] strings; // 不合规的
}

public class A {
  public static String [] strings1 = {"first","second"};  // 不合规的
  public static String [] strings2 = {"first","second"};  // 不合规的
  public static List<String> strings3 = new ArrayList<>();  // 不合规的
  // ...
}

请详见

  • MITRE, CWE-582 - Array Declared Public, Final, and Static
  • MITRE, CWE-607 - Public Static Final Field References Mutable Object
  • CERT, OBJ01-J. - Limit accessibility of fields
  • CERT, OBJ13-J. - Ensure that references to mutable objects are not exposed
