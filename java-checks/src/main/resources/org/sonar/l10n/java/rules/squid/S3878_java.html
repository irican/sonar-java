创建一个数组只是为了将它作为varargs（ ... ）参数传递而没有意义; varargs is一个数组。 只需直接传递元素即可。 它们将自动合并为一个数组。 顺便传递一个数组，其中 Object ... 预期会使意图模糊不清：数组应该是一个对象还是一个对象集合？

不合规的代码示例

public void callTheThing() {
  //...
  doTheThing(new String[] { "s1", "s2"});  // 不合规：不必要
  doTheThing(new String[12]);  //合规的
  doTheOtherThing(new String[8]);  // 不合规：含糊不清
  // ...
}

public void doTheThing (String ... args) {
  // ...
}

public void doTheOtherThing(Object ... args) {
  // ...
}

合规的解决方案

public void callTheThing() {
  //...
  doTheThing("s1", "s2");
  doTheThing(new String[12]);
  doTheOtherThing((Object[]) new String[8]);
   // ...
}

public void doTheThing (String ... args) {
  // ...
}

public void doTheOtherThing(Object ... args) {
  // ...
}
