使 public 常量为 final 而不是 static final 会导致为类的每个实例复制它的值， 从而毫无用处地增加执行应用程序所需的内存量。

此外，当非-public， final 字段也不是 static时，它意味着不同的实例可以具有不同的值。 但是，在其声明中初始化非-static final 段会强制每个实例具有相同的值。因此，这些字段应该是 static 或在构造函数中初始化。

不合规的代码示例

public class Myclass {
  public final int THRESHOLD = 3;
}

合规的解决方案

public class Myclass {
  public static final int THRESHOLD = 3;    // 合规
}

例外

对于类型不是基本类型或字符串的内部类的final字段，不会报告任何问题。根据Java规范：
