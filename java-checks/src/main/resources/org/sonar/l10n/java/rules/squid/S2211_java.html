共享编码约定允许团队有效协作。 虽然lambda参数的类型是可选的，但无论如何都要指定它们会使代码更清晰，更容易阅读。

不符合规范的代码示例

Arrays.sort(rosterAsArray,
    (a, b) -> {  // 不合规的
        return a.getBirthday().compareTo(b.getBirthday());
    }
);

合规解决方案

Arrays.sort(rosterAsArray,
    (Person a, Person b) -> {
        return a.getBirthday().compareTo(b.getBirthday());
    }
);

例外

当lambda有一个或两个参数且没有块时，这个规则不会引发问题，因为在这些情况下，这些参数被认为更具可读性。

stream.map((a, b) -> a.length); // 合规的
