java.security.SecureRandom 类提供适用于加密的强随机数生成器（RNG）。 但是，以恒定或其他可预测的值播种它会显着削弱它。 通常，依赖 SecureRandom 实现提供的种子会更安全

当使用以下任一种子的种子调用 SecureRandom.setSeed（）或 SecureRandom（byte []）时，此规则会引发问题：

  • a constant
  • System.currentTimeMillis()

不合规的代码示例

SecureRandom sr = new SecureRandom();
sr.setSeed(123456L); // 不合规的
int v = sr.next(32);

sr = new SecureRandom("abcdefghijklmnop".getBytes("us-ascii")); // 不合规的
v = sr.next(32);

合规的解决方案

SecureRandom sr = new SecureRandom();
int v = sr.next(32);
