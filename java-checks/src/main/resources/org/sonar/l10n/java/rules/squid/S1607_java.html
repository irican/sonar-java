例如，当测试因基础架构问题而失败时，您可能希望暂时忽略它。但是，如果没有关于为什么忽略测试的某种注释（notation），它可能永远不会被重新激活。 如果没有对项目的全面了解，这些测试很难解决，并且最终污染他们的项目。

对于每个被忽略的测试，若是没有注释去说明为什么要跳过它，该规则都会报告一个问题。

@Test

  // ...

合规的解决方案

@Test
public void testDoTheThing() {
  // ...
