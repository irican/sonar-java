操作员不必要的重复通常是打字错误。当要写!i时，没有理由写成 !!!i。

另一方面，递增和递减运算符的重复可能是故意的，但是这样做会混淆意思，应该简化。

这条规则对以下序列会引发问题: !， ~， -， 和 +。

不合规的代码示例

int i = 1;

int j = - - -i;  // 不合规；只需要 -i
int k = ~~~i;    // 不合规；同样的 i
int m = + +i;    // 不合规；操作符在这无用

boolean b = false;
boolean c = !!!b;   // 不合规

合规的解决方案

int i =  1;

int j = -i;
int k = ~i;
int m =  i;

boolean b = false;
boolean c = !b;

例外

使用~~编译GWT时的溢出处理将被忽略
