使用 @Override 注释有两个原因:

  • 如果带注释的方法实际上没有重写任何内容(如拼写错误)，则编译器会发出警告。
  • 它通过明确表示方法被重写来提高源代码的可读性。

不合规的代码示例

class ParentClass {
  public boolean doSomething(){...}
}
class FirstChildClass extends ParentClass {
  public boolean doSomething(){...}  // 不合规
}

合规的解决方案

class ParentClass {
  public boolean doSomething(){...}
}
class FirstChildClass extends ParentClass {
  @Override
  public boolean doSomething(){...}  // 合规
}

例外

从 Object 重写方法时，比如 toString()， hashCode() ……，可以放宽此规则。
