@RequestMapping 方法默认处理所有匹配的请求。 这意味着你想要的方法 POST-ed 也可以通过GET, 调用，从而允许黑客不恰当地调用该方法。 例如，可以像这样调用“transferFunds”方法：<img src="http://bank.com/actions/transferFunds?reciepientRouting=000000&receipientAccount=11111111&amount=200.00" width="1" height="1"/>

出于这个原因，您应该始终明确列出您期望@RequestMapping Java方法的单个HTTP方法 调用。 当缺少 method 时，此规则会引发问题。

不合规的代码示例

@RequestMapping("/greet")  // 不合规的
public String greet(String greetee) {

合规解决方案

  @RequestMapping("/greet", method = GET)
  public String greet(String greetee) {
