当方法调用中的参数名称与方法参数的名称匹配时，它有助于更清晰，更易读的代码。 但是，当名称匹配但以与方法参数不同的顺序传递时，它表示参数顺序中的错误，这可能会导致意外结果。

不合规的代码示例

public double divide(int divisor, int dividend) {
  return divisor/dividend;
}

public void doTheThing() {
  int divisor = 15;
  int dividend = 5;

  double result = divide(dividend, divisor);  // 不合规; 操作成功，但结果出乎意料
  //...
}

合规解决方案

public double divide(int divisor, int dividend) {
  return divisor/dividend;
}

public void doTheThing() {
  int divisor = 15;
  int dividend = 5;

  double result = divide(divisor, dividend);
  //...
}
