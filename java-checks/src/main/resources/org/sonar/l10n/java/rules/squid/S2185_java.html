某些数学运算只是愚蠢而且不应该执行，因为它们的结果是可预测的。

特别是， anyValue % 1 是愚蠢的，因为它总是返回0。

将非浮点值转换为浮点值，然后将其传递给 Math.round， Math.ceil， 或 Math.floor 是愚蠢的， 因为结果将始终是原始值。

这些运算对于任何常量值都是愚蠢的： Math.abs， Math.ceil， Math.floor， Math.rint， Math.round。

在某些常量下，这些运算是愚蠢的:

acos 0.0 or 1.0

不合规的代码示例

public void doMath(int a) {
  double floor = Math.floor((double)a); // 不合规
  double ceiling = Math.ceil(4.2);  // 不合规
  double arcTan = Math.atan(0.0);  // 不合规
}
