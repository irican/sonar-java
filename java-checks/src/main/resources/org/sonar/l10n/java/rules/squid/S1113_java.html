当垃圾收集器（garbage collector）确定没有对该对象的更多引用时，对该对象调用 Object.finalize() 方法。 但绝对不能保证只要一删除对象的最后一个引用，就调用此方法。它可以是几微秒到几分钟之后。因此，当系统资源需要由对象处理时， 最好不要依赖这种异步机制来处理它们。

不合规的代码示例

public class MyClass {
  ...
  protected void finalize() {
    releaseSomeResources();    // 不合规
  }
  ...
}

请参阅

  • CERT, MET12-J. - Do not use finalizers
