盲目地导入包中的所有类会使类命名空间变得混乱，并且可能导致具有相同名称的不同包中的类之间发生冲突。 另一方面，特别列出必要的类可以避免这个问题，并明确需要哪些版本。

不合规的代码示例

import java.sql.*; // 不合规
import java.util.*; // 不合规

private Date date; // 日期类存在于java.sql和java.util中。 这是哪一个？

合规解决方案

import java.sql.Date;
import java.util.List;
import java.util.ArrayList;

private Date date;

例外

Static imports are ignored by this rule. E.G.

import static java.lang.Math.*;
