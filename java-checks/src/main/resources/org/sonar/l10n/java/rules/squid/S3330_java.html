HttpOnly cookie属性告诉浏览器阻止客户端脚本读取具有该属性的cookie，并且它的使用可以在很大程度上抵御跨站点脚本（XSS）攻击。 因此，作为预防措施，默认情况下应在服务器端设置的所有cookie上设置该属性，例如会话ID cookie。

在实施跨站点请求伪造（XSRF）保护时，应在第一个HTTP GET请求上创建一个通常名为XSRF-TOKEN的JavaScript可读会话cookie。 对于这样的cookie， HttpOnly 属性应该设置为“false”。

设置属性可以通过编程方式完成，也可以通过配置文件全局完成。

不合规的代码示例

Cookie cookie = new Cookie("myCookieName", value); //不合规; 默认情况下，cookie.isHttpOnly（）返回false

合规解决方案

Cookie cookie = new Cookie("myCookieName", value);
cookie.setHttpOnly(true); // 合规的
