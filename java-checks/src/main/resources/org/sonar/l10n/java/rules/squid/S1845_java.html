查看类中的方法集（包括超类方法），以及查找仅通过大小写来区分的两个方法或字段会使该类的用户感到困惑。 同样令人困惑的是， 只有大写字母或方法和字段不同的方法和字段具有完全相同的名称和可见性。

在这种方法的情况下，最初的开发人员可能犯了一个错误，他们打算重写一个超类方法，但是添加了一个几乎同名的新方法。

否则，这种情况只表明命名不当。方法名应该是面向操作的，因此包含一个动词，在方法和成员都具有相同的名称(无论大小写是否不同)的情况下， 这是不可能的。然而，重命名公共方法可能会对调用者造成干扰。因此，建议重命名成员。

不合规的代码示例

public class Car{

  public DriveTrain drive;

  public void tearDown(){...}

  public void drive() {...}  // 不合规； 重复的字段名
}

public class MyCar extends Car{
  public void teardown(){...}  // 不合规； 不是一个重写。 真的是想要的吗?

  public void drivefast(){...}

  public void driveFast(){...} //嗯?
}

合规的解决方案

public class Car{

  private DriveTrain drive;

  public void tearDown(){...}

  public void drive() {...}  // 字段可见性降低
}

public class MyCar extends Car{
  @Override
  public void tearDown(){...}

  public void drivefast(){...}

  public void driveReallyFast(){...}

}
