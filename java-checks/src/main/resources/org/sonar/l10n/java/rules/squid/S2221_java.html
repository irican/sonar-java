捕获 Exception 似乎是处理多个可能异常的有效方法。不幸的是，它会捕获所有异常类型，包括已检查和运行时异常，从而构建过于宽泛的网络。实际上，开发人员是否真的想捕获运行时异常？为了防止任何误解，如果确实要捕获已检查和运行时异常，则应在catch 子句中明确列出它们。

就是说如果 try块中的方法没有明显地（显式地）抛出Exception ，则此规则会引发问题。

不合规的代码示例

try {
  // do something that might throw an UnsupportedDataTypeException or UnsupportedEncodingException
} catch (Exception e) { // 不合规的
  // log exception ...
}

合规的解决方案

try {
  // do something
} catch (UnsupportedEncodingException|UnsupportedDataTypeException|RuntimeException e) {
  // log exception ...
}

or if runtime exceptions should not be caught:

try {
  // do something
} catch (UnsupportedEncodingException|UnsupportedDataTypeException e) {
  // log exception ...
}
