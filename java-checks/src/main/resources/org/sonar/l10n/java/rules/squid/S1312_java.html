无论使用什么日志框架（logback， log4j， commons-logging， java.util.logging， ...），记录器（loggers）应该是：

  • private：永远不能在其父类之外访问。 如果另一个类需要记录某些东西，它应该实例化它自己的记录器。
  • static： 不依赖于类(对象)的实例。在记录某些内容时，当然可以在消息中提供上下文信息，但是应该在类级别创建日志程序， 以防止随着每个对象一起创建日志程序。
  • final： 每个类中只创建一次。

不合规的代码示例

默认正则表达式为 LOG(?:GER)?：

public Logger logger = LoggerFactory.getLogger(Foo.class);  // 不合规

合规的解决方案

private static final Logger LOGGER = LoggerFactory.getLogger(Foo.class);

例外

org.apache.maven.plugin.logging.Log 类型的变量将被忽略。
