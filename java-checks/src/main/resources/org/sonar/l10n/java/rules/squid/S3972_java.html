当每个语句都有自己的行时，代码最清晰。 尽管如此，在同一行上组合 if 及其产生的 then 语句是一种常见的模式。 但是，如果 if 与前面的 else 或 else if 中的结束} 放在同一行上，那么它是一个错误 - else 缺失 - 或者是对未来错误的邀请，因为维护者无法理解这两个语句是未连接的。

不符合规范的代码示例

if (condition1) {
  // ...
} if (condition2) {  // 不合规的
  //...
}

合规解决方案

if (condition1) {
  // ...
} else if (condition2) {
  //...
}

Or

if (condition1) {
  // ...
}

if (condition2) {
  //...
}
