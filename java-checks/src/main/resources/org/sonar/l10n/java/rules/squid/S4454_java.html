通过契约，来自 java.lang.Object 的 equals(Object)方法应该接受 null 参数。在所有其他情况中， null 情况甚至在Object.equals(...) Javadoc中明确详细说明“_对于任何非空引用值x， x.equals（null）应返回false ._”

假设equals 的参数始终为非null，并且使用注释强制执行该假设不仅违反了equals的合同，而且随着课程的使用随着时间的推移而发展，也可能在未来引起问题。

当覆盖 equals 方法且其参数使用任何 @Nonnull注释时，该规则会引发问题

不符合规范的代码示例

public boolean equals(@javax.annotation.Nonnull Object obj) { // 不合规的
  // ...
}

合规解决方案

public boolean equals(Object obj) {
  if (obj == null) {
    return false;
  }
  // ...
}
