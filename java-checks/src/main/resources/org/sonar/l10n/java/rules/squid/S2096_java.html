没有理由使用 main 方法来 throw 任何东西。毕竟，有什么 catch 它呢？

相反，该方法本身应该优雅地处理可能出现的任何异常，附加尽可能多的上下文信息，执行任何必要的日志记录或用户通信。 并且如果有必要的话，使用非零（即失败）退出代码来 exit 。

不合规的代码示例

public static void main(String args[]) throws Exception { // 不合规
  doSomething();

合规的解决方案

public static void main(String args[]) {
 try {
    doSomething();
  } catch (Throwable t) {
    log.error(t);
    System.exit(1);  // 默认退出代码0表示成功。非零值表示失败。
  }
}
