Cookie的域名指定哪些网站应该能够阅读它。如果留空，浏览器应该只将cookie发送到与发送域完全匹配的网站。例如，如果cookie是由lovely.dream.com设置的，那么它应该只能由该域读取，而不能由 nightmare.com 或甚至 strange.dream.com. 如果您想允许cookie的子域访问，您可以通过在cookie的域前添加一个点来指定它，如下所示： .dream.com. 但cookie域应始终使用至少两个级别

Cookie域可以通过编程方式或通过配置进行设置。当任何cookie域设置为单个级别时，此规则会引发问题，如.com.

不符合规范的代码示例

Cookie myCookie = new Cookie("name", "val");
myCookie.setDomain(".com"); // 不合规的
java.net.HttpCookie myOtherCookie = new java.net.HttpCookie("name", "val");
myOtherCookie.setDomain(".com"); // 不合规的

Compliant Solution

Cookie myCookie = new Cookie("name", "val"); // 合规;默认情况下，cookie仅返回发送它们的服务器。

// 或者

Cookie myCookie = new Cookie("name", "val");
myCookie.setDomain(".myDomain.com"); //合规的

java.net.HttpCookie myOtherCookie = new java.net.HttpCookie("name", "val");
myOtherCookie.setDomain(".myDomain.com"); // 合规的
