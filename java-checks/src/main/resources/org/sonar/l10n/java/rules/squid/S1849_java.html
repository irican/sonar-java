调用 Iterator.hasNext() 不应该有任何副作用（side effect），因此不应该更改迭代器的状态。 Iterator.next() 将迭代器向前推进一项。因此，在 Iterator.hasNext()中调用它会破坏 hasNext() 合约，并导致生产中出现意外行为。

不合规的代码示例

public class FibonacciIterator implements Iterator<Integer>{
...
@Override
public boolean hasNext() {
  if(next() != null) {
    return true;
  }
  return false;
}
...
}
